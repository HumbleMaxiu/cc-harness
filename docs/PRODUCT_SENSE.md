# Product Sense — cc-harness

## 产品为谁服务

### 用户画像

| Persona | 需求 |
|---------|------|
| **AI-First 开发者** | 在项目中引入结构化的 AI 协作流程，提升代码质量和一致性 |
| **Harness 新用户** | 不知道什么是 harness，希望快速起步而非阅读大量文档 |
| **AI 工具探索者** | 已经在用 Cursor/Windsurf/Codex，希望迁移或整合 |
| **团队 lead** | 为团队统一 AI 工具标准，建立交接规范 |

### 非目标（Non-goals）

- ❌ 不提供运行时执行引擎（依赖 Claude Code 环境）
- ❌ 不提供 UI 或 Web 界面
- ❌ 不强制特定的编程语言或框架

### UX 原则

1. **越少越好** — Scaffold 生成的文件数最小化，但每个都有明确目的。
2. **渐进式披露** — 用户先看到 AGENTS.md 的简短索引；需要时再深入 docs/。
3. **上下文自解释** — 每个文档文件的开头有一句话描述其用途。
4. **跨平台兼容** — Bridge 文件支持多种 agent IDE，最大化适用范围。

### 什么是"好"的 harness

- 新开发者 clone 后，AI agent 能独立完成端到端任务
- 所有设计决策和执行状态都在 Git 中，无需外部工具或记忆
- Agent 工作流有记录（交接文档），人类可以审查和接管
