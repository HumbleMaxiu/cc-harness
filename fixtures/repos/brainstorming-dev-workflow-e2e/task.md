# brainstorming-dev-workflow-e2e

## Task

Start with a small ambiguous request, use `brainstorming` to shape the approach, turn it into a plan, then complete it through `dev-workflow` in Skill mode.

The grader expects these checkpoints:

1. Brainstorming explores multiple options and a recommendation.
2. An exec plan exists before implementation.
3. `Skill Workflow Record` includes `Mode Decision`, `Execution`, `Self Review`, `Verification`, `Doc Sync`, and `Final Summary`.
4. The final run trace shows the workflow reached `Done`.
