# greenfield-scaffold behavior eval

## Task

A fresh repository has just adopted `cc-harness`.

Grade the resulting scaffolded repo for minimum usability:

1. Core harness docs exist.
2. `AGENTS.md` is concise and includes `How to use this harness`.
3. Memory, feedback, exec-plan, and product-spec indexes exist.
4. The scaffold is linked enough that a new session can recover context.

## What the grader looks for

- Presence of the minimum harness file set for a greenfield repo.
- `AGENTS.md` includes workflow entrypoints and stays compact.
- Core index/navigation files exist for docs, exec plans, product specs, and memory.
