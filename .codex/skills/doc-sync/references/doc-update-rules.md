# Doc Update Rules

## 基本规则

- 优先做与当前变更直接相关的最小更新
- 保持现有文档结构、语气和信息层级一致
- 不要为了“显得同步了”而做无意义改写

## 新增文档时

- 补齐相关 index
- 如有入口导航，也同步 `AGENTS.md` 或对应入口页
- 必要时补交叉链接

## 无需修改时

- 明确记录到 `reviewed_no_change`
- 最好附带一句原因，避免后续重复检查同一问题

## 暂不处理时

- 记录到 `follow_up_needed`
- 说明为什么这次不适合立即更新

## 架构相关文档

当变更涉及角色边界、workflow 结构、系统职责或跨模块关系时：

- 检查 `ARCHITECTURE.md`
- 检查相关 `docs/design-docs/*.md`
- 若暂不更新，必须给出明确理由
