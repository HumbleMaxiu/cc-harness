# 自动评估触发条件与决策逻辑

本文件定义何时需要调用 `agents-md-guide` 的各工作流，以及如何做出决策。

---

## 触发条件清单

### 条件 1：新子目录创建

**检测方式**：Agent 在工作过程中发现新目录（如 `packages/xxx`、`src/xxx/` 等）

**评估决策树**：

```
新子目录是否存在？
├── 否 → 不处理
└── 是 → 满足以下所有条件？
    ├── 子模块有独立职责边界？（能在 1 句话内描述）
    ├── 子模块有特有的约定？（3+ 条模型无法自行推断的规则）
    ├── 子模块有 5+ 个文件？
    └── Agent 可能独立处理此模块？
    ├── 全部满足 → 创建子目录 AGENTS.md + 更新索引
    └── 任一不满足 → 不创建，记录判断理由
```

**决策示例**：

```markdown
## 评估：src/utils/

- [ ] 独立职责：❌ utils/ 是通用工具集合，无法一句话描述
- [ ] 特有约定：❌ 通用工具约定，模型可自行推断
- [ ] 5+ 文件：✅ 有 12 个文件
- [ ] 独立处理：❌ 通常与其他模块配合使用

结论：不创建 AGENTS.md。
理由：utils/ 是通用模块，不满足独立职责边界条件。
```

### 条件 2：docs/ 目录结构变化

**检测方式**：docs/ 下新增、删除或重命名文件

**评估决策树**：

```
docs/ 结构变化？
├── 无对应 AGENTS.md 条目 → 添加到文档地图
├── 有对应条目但路径变化 → 更新路径
├── 文档被删除 → 从文档地图移除，提示用户
└── 无变化 → 不处理
```

---

### 条件 3：AGENTS.md 长度超标

**检测方式**：读取 AGENTS.md 行数

**评估决策树**：

```
AGENTS.md 行数 > 300？
├── 是 → 识别可拆分章节
│   ├── 详细约定（可归类为编码规范）→ docs/conventions.md
│   ├── 测试相关 → docs/testing.md
│   ├── 部署相关 → docs/deployment.md
│   └── 其他 → 按主题归类
└── 否 → 检查其他条件
```

---

### 条件 4：AGENTS.md 过时

**检测方式**：查找最后验证日期

**评估决策树**：

```
AGENTS.md 最后验证日期？
├── 无日期 → 添加 "[最后验证: YYYY-MM-DD]"
├── 日期 > 6 个月前 → 触发时效性检查
│   ├── 核心内容仍正确 → 更新日期
│   └── 内容需要更新 → 按 P0/P1/P2 改进
└── 日期 < 6 个月 → 不处理
```

---

### 条件 5：Agent 表现下降

**检测方式**：用户反馈 Agent 输出质量下降

**评估决策树**：

```
用户反馈 Agent 表现下降？
├── Agent 忽略已知约定？→ 检查 AGENTS.md 是否存在且清晰
├── Agent 做出错误假设？→ 检查是否缺少关键约束
├── Agent 输出不一致？→ 检查是否缺少工作状态管理
├── Agent 上下文丢失？→ 检查是否需要 progress.md
└── 其他 → 读取 references/07-diagnosis.md（Harness Engineering）
```

## 决策记录模板

每次自动触发时，记录决策理由：

```markdown
## [YYYY-MM-DD] AGENTS.md 评估记录

### 触发条件
[条件类型]

### 评估结果
[评估过程]

### 决策
[创建/不创建/改进]

### 理由
[具体理由]

### 相关文件
- AGENTS.md
- docs/xxx.md（如有变更）
```
