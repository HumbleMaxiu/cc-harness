## 模板结构

```markdown

> 最后更新：YYYY-MM-DD
> 作者：[名字]

[系统是什么，解决什么问题，核心价值]

### 1.2 关键指标
- 当前代码规模：[X 行代码，X 个模块]
- 技术栈版本：[关键依赖版本]
- 部署规模：[用户量/请求量，如适用]

## 2. 架构图

### 2.1 系统组件
```
[组件 A] → [组件 B] → [数据存储]
     ↑
[前端/客户端]
```

### 2.2 数据流
[描述核心请求/响应流程]

## 3. 核心模块

| 模块 | 路径 | 职责 | 依赖关系 |
|------|------|------|----------|

### 3.1 [模块 A]
[模块职责详细描述]

**暴露接口：**
- `[API/方法/事件]`

- `[模块 B]`：用于 [具体用途]

### 4.2 禁止模式
[明确列出不允许的架构模式，如：
- 禁止前端直接访问数据库

## 5. 外部依赖

| 服务 | 用途 | 配置位置 |
|------|------|----------|

## 6. 安全边界

### 6.1 认证与授权

## 7. 可观测性

| 维度 | 实现 | 位置 |
|------|------|------|
| 日志 |Structured JSON logs | stdout → ELK/Loki |

## 渐进披露层级

本模板分为 3 个复杂度层级：

| 层级 | 适用场景 | 包含章节 |
|------|----------|----------|
| L1 简单 | 单模块应用，Solo 开发 | 概述 + 模块 + 边界 |
| L2 中等 | 多模块，有 API 层 | + 架构图 + 数据流 |
| L3 复杂 | 微服务/多团队 | + 安全边界 + 可观测性 |

按项目复杂度选择生成哪些章节。
